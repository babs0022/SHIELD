How It Works

End-to-end private sharing

Shield coordinates a simple, wallet-gated flow between sender and receiver. Every link is bound to a specific EVM-compatible address and time-limited for safety.

Sender

Create a protected share link

1

Upload

Add the content you want to share — either a file or an encrypted text message — directly in your browser.

2

Recipient wallet

Provide the receiver’s EVM-compatible wallet address. The link will only decrypt for this exact address.

3

Access limits

Choose how long the link should stay valid (in seconds) and how many times the content can be opened before it expires.

4

Generate link

Hit the Generate Link button, sign the transaction in your connected wallet, and let Shield encrypt and register the share.

Once confirmed, copy the share link and send it to the intended receiver through any channel you prefer.

Receiver

Prove ownership, then decrypt

1

Open & connect

Open the shared link and connect your wallet in the Shield interface using any supported EVM-compatible wallet.

2

Verify ownership

Click the Verify Wallet Ownership button and approve a gasless signature (followed by a transaction) request in your connected wallet.

Shield checks that the connected wallet matches the address chosen by the sender. If it matches and the link is still valid, the encrypted content is unlocked for you.

3

Access content

View the decrypted file or message and choose whether to copy, download, or save it — all without exposing anything to any centralized server.
